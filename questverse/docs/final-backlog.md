# QuestVerse 最终保留项

> 用途：记录“最终版本需要完成，但不阻塞当前网页端 demo 验收”的内容。
> 当前 Phase 2 验收范围只考虑桌面网页端；移动端、完整无障碍和大型视觉升级进入后续阶段。

## 1. 平台与适配

### 1.1 移动端完整适配

**建议阶段**：Phase 4

当前仅做过 375x667 截图烟测，确认页面可渲染；完整移动端体验暂不作为 Phase 2 验收阻塞项。

最终需要完成：

- 375x667、390x844、430x932 等主流手机视口专项测试；
- 热点点击命中率优化；
- 谜题弹窗移动端滚动、按钮间距、输入框布局优化；
- 隐藏对象谜题在触屏上的命中反馈；
- 横屏/竖屏策略；
- 移动端性能预算。

### 1.2 键盘与无障碍完整验收

**建议阶段**：Phase 3 后半 / Phase 4

Phase 2 保留基础按钮语义；完整无障碍专项放入最终验收。

最终需要完成：

- Tab 顺序梳理；
- Enter/Space 操作一致性；
- 谜题弹窗焦点锁定；
- 关闭弹窗后焦点回到原 hotspot；
- `aria-live` 用于彩蛋、奖励、失败反馈；
- 色彩对比度检查；
- 减少动画模式。

## 2. 视觉与空间升级

### 2.0 第一星球 3D 手势原型后续打磨

**当前状态**：已在 Phase 2.5 原型中完成第一星球 3D 霓虹圣树空间，真实摄像头手势识别可控制旋转与缩放，原有 hotspots / 谜题 / 物品 / 彩蛋交互继续复用。

后续不阻塞当前验收，但最终需要继续打磨：

- 摄像头权限失败、模型下载失败、低性能设备的降级说明；
- 真实玩家手势灵敏度标定；
- 摄像头开启后的隐私提示与状态反馈；
- 更完整的节点遮挡、标签避让与场景层级；
- 低性能设备降级到鼠标/触控拖拽模式。

### 2.1 3D 星球选择界面

**建议阶段**：Phase 3 原型，Phase 4 打磨

先把星球选择页升级成 3D 宇宙地图，每颗星球是可旋转球体。这样能提升“QuestVerse 宇宙感”，但不推翻当前 2D 场景与 hotspot 系统。

最终需要完成：

- Three.js 或 React Three Fiber 技术验证；
- 桌面端 3D 星球选择页；
- 星球 hover/click 命中；
- 低性能设备降级到 2D 卡片；
- 每颗星球独立材质与主题动效。

### 2.2 全 3D 星球探索

**建议阶段**：Phase 4+，暂不进入 Phase 3 主线

如果未来要把游戏做成“像地球一样可在球面探索”的 3D 星球，需要重新设计：

- 球面坐标与热点定位；
- 相机控制；
- 3D 资产生产流程；
- 移动端性能；
- 谜题 UI 与 3D 场景的衔接。

这属于大架构升级，不阻塞当前网页端 demo。

## 3. 氛围与内容增强

### 3.1 第二星球恐怖氛围增强

**建议阶段**：Phase 3 前半

第二星球的主题是噪声、倒放、实体媒介与勇气。可以继续加强 analog horror 氛围，但保持“寻宝解谜”本质，不转成纯恐怖游戏。

最终需要完成：

- 更压迫的灯光闪烁；
- Echo-7 断续、失真、延迟反馈；
- 磁带档案室黑场、雪花点、时间码异常；
- 倒放标签解谜前后的环境变化；
- 失败后更具情绪的反馈文本；
- 可选的低强度音效方案。

暂不建议：

- 频繁 jump scare；
- 血腥或猎奇元素；
- 追逐战；
- 会让玩家死锁或丢失进度的恐怖惩罚。
