## 1. 愿景 (Vision)

**一句话定义**: QuestVerse 是一款以"寻宝解谜 + 彩蛋文化"为核心玩法的浏览器互动叙事游戏，玩家在霓虹宇宙中探索不同主题星球、解开层层谜题、寻找隐藏的"终极彩蛋"。

**灵感来源**: 电影《头号玩家》(Ready Player One, 2018)

**情感目标**:
- **发现感** — 玩家在场景中"看见"平时不会注意的细节
- **怀旧共鸣** — 致敬 80-00 年代游戏/电影/动漫文化
- **解谜成就感** — 通过思考而非操作技巧推进
- **诗意** — 像翻阅一本被遗忘的日记
- **连接** — 玩家与创作者、玩家与玩家之间的隐秘连接

**与同类游戏的差异**:
| 维度 | QuestVerse | 传统密室逃脱 | 点密游戏 | MMO 元宇宙 |
|---|---|---|---|---|
| 平台 | Web 浏览器 | 各种 | Web 浏览器 | 客户端 |
| 规模 | 单人为主 | 单人 | 多人 | 多人 |
| 美术 | 像素+霓虹 | 实景 | 2D 手绘 | 3D 写实 |
| 节奏 | 慢，解谜思考 | 紧张倒计时 | 社交为主 | 长线 |
| 社区 | 隐藏彩蛋发现者 | 无 | 在线 | 实时 |
| 制作成本 | 低（个人项目） | 高 | 中 | 极高 |

## 2. 核心循环 (Core Loop)

```
       发现/触发
        ↓
       探索场景 ──── 找到线索（物品/文字/视觉）
        ↓
       触发谜题
        ↓
       思考解谜 ──── 可请求提示（3 阶段）
        ↓
       解决谜题
        ↓
       获得钥匙碎片 / 解锁新区域
        ↓
       推进到下一星球
        ↓
       循环继续
```

**循环节奏**: 30 秒可探索一屏 → 1-3 分钟可解开一个谜题 → 一颗星球 5-8 个谜题 → 整关卡 45-90 分钟

## 3. 三把钥匙叙事 (Three Keys)

致敬《头号玩家》的三把钥匙——它们不仅是游戏机制，更是**精神成长**的隐喻：

| 钥匙 | 主题 | 谜题类型 | 精神隐喻 | 玩家学会 |
|---|---|---|---|---|
| **铜钥匙 (Copper)** | 逆向思维 | 序列倒序、密码倒推 | "规则不是绝对的" | 重新审视"标准答案" |
| **玉钥匙 (Jade)** | 勇气 | 恐怖谜题、未知探索 | "恐惧是想象的囚徒" | 接受不确定性 |
| **水晶钥匙 (Crystal)** | 智慧 | 知识考验、跨媒介引用 | "学识是积累而非天生" | 文化拼图的快乐 |

**注意**: 三把钥匙是**叙事框架**，不是必须严格三颗星球。  
**第二期再加入** — 一期只有"铜钥匙"完整流程。

## 4. 世界观 (Lore)

### 4.1 时代背景

**2040 年代**。一场全球性的"云端雪崩"摧毁了几乎所有 VR 服务器。人们在碎片化的离线世界中寻找失去的连接。

**QuestVerse** 是一款在灾难前发布的开源单机游戏，由传奇游戏设计师 **A.N. Reverie** (致敬 A.N. Rice) 制作。他将游戏源代码与一段加密信息一同保存在七颗散布全球的离线终端中。

玩家收到一封匿名信函：
> *"如果你正在读这条消息，你应该已经在游戏中了。*
> *七颗世界，七把钥匙，七个真相。"*

### 4.2 七颗星球 (The Seven Worlds)

> 致敬《头号玩家》的 27 分区架构，但缩减为 7 颗主题明确的星球

| # | 星球 | 主题 | 致敬对象 | 颜色 |
|---|---|---|---|---|
| 1 | 霓虹街机厅 (Neon Arcade) | 80s 街机 | Atari 2600 / Pong / Pac-Man | 霓虹粉 |
| 2 | 黑胶唱片室 (Vinyl Vault) | 80s 音乐 | Beatles / Kraftwerk / Joy Division | 暗紫 |
| 3 | 录像带之海 (VHS Sea) | 80s/90s 影视 | Kubrick / Spielberg / Lynch | 深红 |
| 4 | 漫画英雄 (Comic Verse) | 漫画与动画 | DC / Marvel / 大友克洋 | 暖黄 |
| 5 | 像素地下城 (Pixel Dungeon) | 经典 RPG | D&D / 创世纪 / 博德之门 | 锈红 |
| 6 | 摇滚磁场 (Synth Wave) | 合成器浪潮 | 银翼杀手 / 怪奇物语 | 电光青 |
| 7 | 最终沙漏 (Final Hourglass) | 收束地 | 无（原创） | 极光白 |

### 4.3 核心人物

| 名字 | 身份 | 在游戏中的角色 |
|---|---|---|
| **A.N. Reverie** | 传奇游戏设计师 | 死者，留下遗产与线索 |
| **Echo-7** | AI 助手 | 玩家的"系统"语音（致敬《银翼杀手》Roy） |
| **The Curator** | 神秘守门人 | 守护每颗星球的"门" |

## 5. 谜题系统 (Puzzle System)

### 5.1 谜题分类

```
PuzzleType
├── "reverse" (逆向) ──── 铜钥匙
│   ├── PuzzleSequence  (序列倒序)
│   ├── PuzzleCode      (密码倒推)
│
├── "courage" (勇气) ──── 玉钥匙
│   ├── PuzzleHiddenObject (隐藏对象)
│   └── 时间压力机制
│
└── "wisdom" (智慧) ──── 水晶钥匙
    ├── PuzzleRiddle   (谜语)
    ├── PuzzleCode     (输入)
    └── PuzzleLogic    (逻辑)
```

### 5.2 谜题设计原则

1. **第一直觉错误** — 谜题设计成"第一个你想到的答案往往是错的"（致敬《头号玩家》赛车关卡）
2. **多路径提示** — 至少 3 阶段提示，避免完全卡关
3. **解决后回顾** — 解决后弹出"线索连贯性说明"，让玩家理解为什么这是对的
4. **失败无惩罚** — 唯一消耗的是时间，谜题本身无限次重试
5. **彩蛋散落** — 谜题场景中可藏 3-5 个彩蛋

### 5.3 提示系统

```typescript
{
  level: 1, // 视觉提示
  text: "看墙上...",
  helpType: "narrow"
},
{
  level: 2, // 文本提示
  text: "那行文字倒过来读",
  helpType: "direct"
},
{
  level: 3, // 答案方向
  text: "答案是 1979 — Warren Robinett 的《Adventure》",
  helpType: "answer"
}
```

## 6. 进度与存档 (Progress & Save)

### 6.1 玩家存档结构

```typescript
{
  currentPlanetId: string;        // 当前位置
  currentSceneId: string;         // 当前场景
  inventory: string[];            // 物品 ID
  solvedPuzzles: string[];        // 已解谜题
  foundEasterEggs: string[];      // 已发现彩蛋
  flags: Record<string, any>;     // 叙事标记
  storyProgress: number;          // 0-100
  hintsUsed: Record<string, number>;
  playTime: number;               // 秒
}
```

### 6.2 持久化策略

| 阶段 | 方案 |
|---|---|
| **Phase 1 (现在)** | LocalStorage（zundo 中间件） |
| **Phase 2** | LocalStorage + 服务端镜像（防丢失） |
| **Phase 3+** | 用户账号 + 云存档 |

## 8. 玩家旅程 (Player Journey)

### 8.1 第一颗星球（30-45 分钟）

**玩家情绪曲线**:

```
困惑 ──→ 好奇 ──→ 探索 ──→ 挫败 ──→ 第一个提示 ──→ "啊哈！" ──→ 解开谜题
                                                       ↓
                                                  成就感 + 彩蛋奖励
                                                       ↓
                            获得铜钥匙 ──→ 推进到下一颗星球
```

### 8.2 关键体验节拍

| 时间 | 事件 | 玩家情感 |
|---|---|---|
| 0:00 | 进入主页 | 好奇 |
| 0:30 | 进入第一颗星球 | 期待 |
| 1:00 | 看到第一个隐藏彩蛋 | 小惊喜 |
| 3:00 | 触发第一个谜题 | 紧张 |
| 5:00 | 卡住，请求提示 | 略失望 |
| 6:00 | "啊哈" 时刻 | 兴奋 |
| 7:00 | 获得铜钥匙 | 成就感 |
| 9:00 | 看到自己的记忆档案 | 持续满足 |

## 10. 不在范围内 (Out of Scope)

为了保持项目可执行，明确以下**不**做：

- ❌ 多人实时联机
- ❌ 真正的 VR/AR
- ❌ 用户账号系统（阶段 1）
- ❌ 复杂 3D 引擎（three.js 留作选项）
- ❌ 移动端原生应用
- ❌ 真实在线支付
- ❌ 高清 3D 资产（像素风起步）
- ❌ 配音

## 11. 风险与缓解 (Risks)

| 风险 | 影响 | 缓解 |
|---|---|---|
| 一个人做艺术太慢 | 高 | 像素风 + Emoji + 开源素材包 (Kenney.nl) |
| 解谜设计质量低 | 高 | 反复测试，每个谜题至少 5 人试玩 |
| Scope creep | 中 | 严格按 GDD 推进，阶段交付 |
| 6 个月不够 | 中 | 已按阶段分拆，阶段 2 完成即有可玩 demo |
| 玩家卡关流失 | 中 | 提示系统 + 活跃度分析 |
| 内容写不完 | 中 | 模板化谜题生成 + 用户创作（远期） |
